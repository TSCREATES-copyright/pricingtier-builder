## 💡 Why This Exists

Every founder hits the same wall.

You have built the product. The landing page is ready. Investors are asking about monetization. Customers are waiting. And then — the pricing page sits blank.

You open a dozen competitor tabs. You pick a number that feels "about right." You launch. And you never really know if you left money on the table, priced yourself out of conversions, or built a tier structure that nobody understood.

**That is the problem PricingTier Builder solves.**

This is not a calculator. It is not an AI brainstormer that generates random suggestions. It is a **deterministic pricing strategy engine** — consistent, explainable, and fast — that applies proven pricing psychology rules to your specific product type and target market, then hands you a complete 3-tier structure you can ship with confidence.

No hallucinations. No cloud. No guesswork.

> Every recommendation the engine makes can be traced back to a rule. You can see *why* a feature landed in Pro and not Premium. You can see *why* the price gap was widened. That transparency is what makes it a tool you trust.

## 🎯 What It Does

PricingTier Builder takes **five inputs** from you:
```
Product Type  →  Target Market  →  Base Price Range  →  Feature Count  →  Spacing Style
```

And produces a **complete, ready-to-ship pricing strategy:**
```
✅ Basic / Pro / Premium tier structure
✅ Recommended monthly + annual price points
✅ Feature allocation per tier (value-stacked)
✅ Comparison table ready for your landing page
✅ Confidence score with metric breakdown
✅ Inline warnings and optimization suggestions
✅ Export in JSON, Markdown, CSV, or plain copy
```

Everything runs **locally in the browser.** No account. No backend. No data leaves your machine.

## ✨ Key Features

Every feature exists to move you closer to a pricing page that converts. No bloat. No filler.

<br />

### 🛠 Core Customization Engine

**Product Type Selection**
Tailor the logic for B2B SaaS, Creator Tools, Developer Tools, Agency Services, or Local Business Software.
→ _The engine adjusts anchor pricing, spacing multipliers, and feature heuristics based on your product category._

**Target Market Profiles**
Choose your audience: Enterprise, SMBs, Indie Hackers, Creators, Agencies, or Developer teams.
→ _Market elasticity profiles determine how aggressively to space tiers and how to frame value._

**Dynamic Base Pricing**
Set your entry-level price preference and the engine intelligently calculates the full tier stack.
→ _No more picking Pro and Premium prices manually — the math is done for you._

**Algorithmic Price Spacing**
Choose between Conservative (1.8x), Geometric (2.2x), or Aggressive (2.8x) multipliers between tiers.
→ _Controls how hard your pricing pushes users toward the mid and top tiers._

<br />

### 🧠 Pricing Psychology Engine

**Charm Pricing Toggle**
Switch between clean prices ($49) and psychologically optimized charm pricing ($47.99) with one click.
→ _Proven to reduce perceived price resistance on mass-market and consumer-leaning products._

**Annual Discount Modeling**
Preview monthly vs. annual price pairs with a configurable discount percentage (default: 20%).
→ _Instantly see how annual pricing affects your MRR/ARR projections and tier attractiveness._

**Premium Exclusivity Slider**
A 1–10 slider that pushes high-value features progressively into higher tiers.
→ _Dial in just enough upgrade pressure without making lower tiers feel useless._

**Anti-Cannibalization Guard**
Automatically enforces rules that prevent Basic from being too generous or Pro from overlapping with Premium.
→ _Keeps your tier structure clean so customers always have a clear reason to upgrade._

<br />

### 📊 Real-Time Strategy Scoring

**Confidence Score Engine**
Evaluates your configuration and outputs a composite score out of 100 with a full metric breakdown.
→ _Know whether your structure is strong before you ship it, not after._

**Five-Dimension Metric Analysis**

| Metric | What It Measures |
|---|---|
| 🎯 **Simplicity** | Is the tier structure easy for a visitor to understand in 5 seconds? |
| ⬆️ **Upgrade Pressure** | Does each tier meaningfully incentivize moving to the next? |
| 💰 **Margin Potential** | Are your highest-value features gated behind the highest tiers? |
| 🔀 **Differentiation** | Is each plan clearly different enough to stand on its own? |
| 🌍 **Market Fit** | Does the pricing align with standard expectations for your market? |

**Inline Optimization Warnings**
The engine flags structural problems in real time — before you export.
→ _"Pro is too weak," "Basic is too generous," "Premium gap is too narrow" — specific, actionable, instant._

<br />

### 📤 Export & Workspace

**Multi-Format Export**
Export your finalized pricing matrix as structured JSON, plain text copy, Markdown table, or CSV.
→ _Paste directly into Notion, your landing page, Stripe, or a dev handoff._

**Version History**
Every meaningful change creates a snapshot. Compare your current structure to a previous version at any time.
→ _Pricing is not a one-time decision. This turns PricingTier Builder into a pricing lab you return to._

**Saved Projects**
Save multiple pricing experiments by product name and revisit them whenever your strategy changes.
→ _Build a library of pricing versions across products, markets, and launch phases._

**Localization (i18n) Export**
Export your pricing structure copy in English (EN), Spanish (ES), and French (FR).
→ _Reduce friction for international pricing pages and multilingual teams._

## 🔄 How It Works

Getting from zero to a shippable pricing structure takes under 10 seconds.

<br />
```
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│   STEP 1          STEP 2          STEP 3          STEP 4        │
│                                                                 │
│  ┌────────┐     ┌────────┐     ┌────────┐     ┌────────┐       │
│  │ Select │────▶│ Set    │────▶│ Engine │────▶│ Refine │       │
│  │ Inputs │     │ Prefs  │     │ Runs   │     │ & Ship │       │
│  └────────┘     └────────┘     └────────┘     └────────┘       │
│                                                                 │
│  Product type    Base price     3-tier output  Edit, score,     │
│  Target market   Spacing mode   Feature alloc  export, save     │
│  Feature count   Annual disc.   Score + notes  version, reuse   │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

<br />

**Step 1 — Describe Your Product**
Select your product type and target market. These two inputs determine the anchor price range, spacing multipliers, and feature allocation heuristics. No free-form text. No ambiguity.

**Step 2 — Configure Your Preferences**
Set your base price preference, choose a pricing style (Conservative / Geometric / Aggressive), set your feature count, and adjust the Premium Exclusivity Slider.

**Step 3 — Engine Generates Your Tiers**
The deterministic engine runs instantly — no API calls, no loading states. You see three fully-formed pricing cards (Basic / Pro / Premium) with prices, features, labels, and a confidence score, all computed locally.

**Step 4 — Refine, Score, Export**
Adjust any input and watch the engine recalculate in real time. When the score is strong and the warnings are clear, export in your preferred format and ship.

## 🏗 Project Architecture

PricingTier Builder follows a strict separation of concerns: **React handles the UI; the `/systems/` modules handle all pricing logic.**

No pricing decisions live inside React components. The engine is fully portable, testable, and independent of the rendering layer.

<br />
```
┌────────────────────────────────────────────────────────────────────┐
│                        PricingTier Builder                         │
├───────────────────────┬────────────────────────────────────────────┤
│     REACT LAYER       │           SYSTEMS LAYER                    │
│  (UI + Interaction)   │       (Deterministic Engine)               │
├───────────────────────┼────────────────────────────────────────────┤
│                       │                                            │
│  AppShell             │  coreEngine.ts                             │
│  SidebarInputs        │    └─ inputNormalizer.ts                   │
│  TierWorkspace        │    └─ pricingTierEngine.ts                 │
│  AdjustmentPanel      │    └─ featureAllocator.ts                  │
│  ComparisonPanel      │    └─ scoringEngine.ts                     │
│  ExportPanel          │    └─ storageManager.ts                    │
│  ProjectLibrary       │                                            │
│                       │  All logic: deterministic, testable,       │
│  State via Context    │  no network calls, instant output.         │
│  + Reducer / Zustand  │                                            │
│                       │                                            │
└───────────────────────┴────────────────────────────────────────────┘
```

<br />

### Architecture Principles

- **Deterministic by design** — The same inputs always produce the same output. No randomness, no AI nondeterminism, no surprises.
- **Zero network dependency** — Every calculation happens locally. No API calls during computation.
- **Logic outside React** — Components are display layers. Business logic lives in `/systems/`.
- **Explainable outputs** — Every recommendation can be traced back to a rule in the engine.
- **Separation of state** — Global state (project, inputs, output) is distinct from local UI state (panel open/close, hover, modal).

## ⚙️ Core Engine Logic

The engine is the product. Here is exactly how it works.

<br />

### Pipeline Overview
```
User Input
    │
    ▼
inputNormalizer.ts
  → Validates and maps input to structured engine config
  → Assigns market profile (elasticity, anchor range, multiplier set)
    │
    ▼
pricingTierEngine.ts
  → Selects anchor price from market profile + base preference
  → Applies spacing multipliers (conservative / geometric / aggressive)
  → Rounds prices using market-appropriate rounding rules
  → Generates Basic, Pro, Premium price points
    │
    ▼
featureAllocator.ts
  → Distributes features using value-stacking heuristics
  → Enforces anti-cannibalization rules
  → Applies premium exclusivity settings
  → Assigns upgrade triggers to Pro tier
    │
    ▼
scoringEngine.ts
  → Scores each dimension (simplicity, upgrade pressure, etc.)
  → Combines into a weighted composite confidence score
  → Generates per-tier warnings and optimization notes
    │
    ▼
outputFormatter.ts
  → Formats result into tier cards, comparison rows, export data
    │
    ▼
React UI renders instantly
```

<br />

### Price Spacing Logic
```typescript
// Conservative: 1.8x / 2.2x
// Geometric:    2.2x / 2.6x  (default)
// Aggressive:   2.8x / 3.2x

const generateTierPrices = (
  anchor: number,
  multiplierA: number,
  multiplierB: number
): TierPrices => ({
  basic:   roundPricing(anchor),
  pro:     roundPricing(anchor * multiplierA),
  premium: roundPricing(anchor * multiplierA * multiplierB),
});

// Example (Geometric, anchor = $19):
// Basic:   $19
// Pro:     $19 × 2.2 = $41.80 → rounded to $49
// Premium: $49 × 2.6 = $127.40 → rounded to $129
```

<br />

### Market Anchor Ranges

| Market Type | Anchor Range | Typical Basic Price |
|---|---|---|

const SCORE_WEIGHTS = {
  simplicity:       0.25,  // Is it easy to understand?
  upgradePressure:  0.25,  // Is there a reason to move up?
  marginPotential:  0.20,  // Are premium features well-gated?
  differentiation:  0.15,  // Is each tier meaningfully distinct?
  marketFit:        0.15,  // Does the pricing match market norms?
};

const confidenceScore = Object.entries(SCORE_WEIGHTS).reduce(
  (total, [key, weight]) => total + scores[key] * weight * 100,
  0
);

## 🗄 Storage Strategy

Everything stays local. No account required. No data leaves the browser.

<br />

| Data Type | Storage Layer | Why |
|---|---|---|
| Current project ID | `localStorage` | Fast, tiny value |
| User settings + preferences | `localStorage` | Instant access on load |
| Feature flags + unlock state | `localStorage` | Simple boolean reads |
| Last-used inputs | `localStorage` | Pre-fill on return visit |
| Saved pricing projects | `IndexedDB` | Large, structured data |
| Version history snapshots | `IndexedDB` | Multi-record persistence |
| Template library | `IndexedDB` | Reusable, growing over time |
| Export logs | `IndexedDB` | Optional audit trail |

<br />

### Unlock State (Free → Pro)
```typescript
type UnlockState = {
  isPro:        boolean;
  licenseKey?:  string;
  unlockedAt?:  number;
};
```

Pro unlock is handled client-side via a license key entry flow. No backend, no auth, no session tokens. The unlock state is persisted in `localStorage` and checked at feature gate boundaries.

## 🤝 Contributing

Contributions are welcome and appreciated. PricingTier Builder is a focused tool — PRs that add complexity without adding user value will not be merged, but improvements to the engine logic, new market profiles, additional export formats, and UI refinements are all high-priority.

<br />

### How to Contribute

**1. Fork the repository**
```bash
git fork https://github.com/yourusername/pricingtier-builder.git
```

**2. Create a feature branch**
```bash
git checkout -b feature/your-feature-name
```

**3. Make your changes**

Follow the existing code conventions. Keep engine logic in `/systems/`. Keep React components as display layers. Run the linter before committing.
```bash
npm run lint
npm run test
```

**4. Commit with a clear message**
```bash
git commit -m "feat: add agency market profile with higher anchor range"
```

**5. Open a Pull Request**

Describe what you changed, why you changed it, and how to test it. Link any relevant issues.

<br />

### Contribution Guidelines

- **Engine changes** require passing unit tests and no regression in the scoring engine output
- **New market profiles** should include an anchor range, multiplier set, and a representative example output in the PR description
- **UI changes** should not increase the component count unnecessarily — prefer extending existing components
- **For major features**, open an issue first to align on scope before writing code
- All contributors are expected to be respectful and constructive

<br />

### Good First Issues

Looking for somewhere to start? Check issues labeled [`good first issue`](../../issues?q=is%3Aissue+is%3Aopen+label%3A%22good+first+issue%22).

Ideas that are always welcome:
